## 📖 이것은 무엇인가요?

웹 브라우저(구글 크롬처럼 웹사이트를 방문할 수 있게 해주는 프로그램)를 열 때마다 **시작 페이지**(브라우저가 열리면 가장 먼저 보이는 화면)가 나타납니다. 보통은 구글 검색 화면이거나, 브라우저가 알아서 정해 준 페이지가 표시되죠.

**이 프로젝트는 그 시작 페이지를 나만의 대시보드로 바꿔 줍니다** -- 즐겨찾기(자주 방문하는 웹사이트 목록)를 정리하고, 메모를 쓰고, 할 일 목록을 관리하고, 습관을 추적하는 등 다양한 기능을 갖춘 아름다운 개인 홈페이지입니다.

대시보드는 작은 **서버**(컴퓨터 뒤에서 조용히 돌아가는 작은 프로그램, 마치 요청을 기다리는 충실한 비서 같은 존재)로 실행됩니다. 이 서버는 **Node.js**(아래 요구 사항 섹션에서 자세히 설명)라는 엔진(engine)을 사용합니다. 데이터는 철저히 비공개로 유지됩니다 -- 어떤 회사나 인터넷 서버로도 데이터가 전송되지 않습니다. 코딩을 할 줄 몰라도 됩니다. 복잡한 소프트웨어를 설치할 필요도 없습니다. 설치 파일 하나만 실행하면 끝입니다.

> **핵심 세 가지:**
> - **코딩 지식이 전혀 필요 없습니다** -- 파일 하나를 더블클릭하면 설치 끝 (자연어로만 "바이브 코딩"하는 분들도 포함됩니다)
> - **데이터는 내 컴퓨터에만 저장됩니다** -- 외부 서버로 절대 전송되지 않습니다
> - **추가 소프트웨어 설치가 필요 없습니다** -- 필요한 것은 자동으로 다운로드하고, 독립적으로 실행됩니다

## ✨ 기능

### 🔖 북마크 & 정리

- **북마크 카드** -- 즐겨찾는 웹사이트를 깔끔한 카테고리(업무, SNS, 쇼핑 등)로 분류합니다
- **여러 페이지** -- 북마크가 많을 때 별도의 탭으로 나눌 수 있습니다 (예: "업무", "개인", "공부")
- **통합 검색** -- 북마크, 메모, 할 일, 일정을 한 번의 키 입력으로 한꺼번에 검색합니다
- **스마트 붙여넣기** -- 웹사이트 주소(URL)를 복사한 뒤 페이지 아무 곳에서 Ctrl+V를 누르면 카테고리 선택 창이 자동으로 나타납니다
- **드래그 앤 드롭** -- 크롬 주소창에서 링크를 카드 위로 끌어다 놓으면 바로 추가됩니다
- **죽은 링크 감지** -- 더 이상 작동하지 않는 북마크를 자동으로 찾아냅니다
- **다중 선택** -- Ctrl 키를 누른 채 여러 북마크를 클릭해서 한꺼번에 이동하거나 삭제할 수 있습니다
- **상단 고정** -- 가장 중요한 북마크를 고정하면 카드 맨 위에 항상 표시됩니다

### 🔍 검색

- **웹 검색** -- 상단 검색창에 입력하고 Enter를 누르면 웹 검색이 실행됩니다
- **다중 검색 엔진** -- 구글, 유튜브, 네이버 등을 클릭 한 번으로 전환할 수 있습니다
- **통합 검색 (스포트라이트)** -- `F` 키를 누르면 북마크, 할 일, 메모, 캘린더 일정을 한꺼번에 검색합니다
- **한국어 초성 검색** -- 초성만 입력해도 (예: "ㄴㅇㅂ") "네이버"처럼 일치하는 북마크가 나타납니다
- **키워드 단축키** -- `yt 고양이`를 입력하면 유튜브에서 "고양이"를 바로 검색합니다
- **명령 모드** -- 통합 검색에서 `>settings`, `>theme`, `>timer` 등을 입력하면 명령이 즉시 실행됩니다

### ✅ 생산성 도구

- **할 일 목록** -- 우선순위(높음/중간/낮음), 태그, 마감일, 반복 일정을 지원하는 작업 목록; 드래그로 순서 변경 가능; 완료된 항목은 자동으로 하단으로 이동
- **메모** -- 간단한 텍스트 메모를 위한 메모 카드를 원하는 만큼 만들 수 있습니다 (마치 일기장(diary)처럼 여러 권을 둘 수 있습니다)
- **캘린더** -- 월간 또는 주간 보기로 일정을 만들고 브라우저 알림을 받을 수 있습니다
- **뽀모도로 타이머** -- 25분 집중 타이머와 선택적 5분 휴식 자동 반복 (자동 세션으로 사이클이 계속됩니다)
- **습관 추적기** -- 매일 자정에 초기화되는 일일 체크리스트

### 🎨 외관 & 사용자 정의

- **다크 / 라이트 테마** -- 클릭 한 번으로 전환 (부드러운 원형 전환 애니메이션), 또는 컴퓨터 설정을 자동으로 따르기
- **6가지 강조 색상** -- 파랑, 보라, 로즈, 에메랄드, 앰버, 시안
- **사용자 지정 배경** -- 자신의 이미지를 업로드; 이미지 1장이면 고정 배경, 2장 이상이면 슬라이드쇼 (ON/OFF 토글, 간격 조절, 수동 이전/다음 버튼)
- **글래스모피즘 UI** -- 카드 투명도(블러 슬라이더)와 불투명도(오파시티 슬라이더)를 조절하여 현대적인 반투명 유리 효과를 연출
- **레이아웃 프리셋** -- 4가지 유형: 기본, 컴팩트, 와이드, 매거진
- **사용자 정의 CSS** -- 고급 사용자를 위한 자체 스타일링 규칙 작성 기능

### 🌤️ 날씨 & 시계

- **날씨 위젯** -- [OpenWeatherMap](https://openweathermap.org/api)을 사용하여 현재 도시의 날씨를 표시합니다 (무료 API 키 필요 -- 아래에서 설명)
- **세계 시계** -- 대시보드 상단에 여러 시간대를 표시합니다 (예: 뉴욕, 런던, 도쿄)

### 📆 D-Day 카운트다운

- "D-7", "D-3", "D-Day", "D+1" 스타일로 중요한 날짜를 추적합니다
- 대시보드 상단에 표시되어 빠르게 확인할 수 있습니다
- 지난 날짜는 며칠이 경과했는지 보여줍니다

### 🔒 안전 & 개인정보 보호

- **100% 로컬** -- 데이터가 절대 컴퓨터 밖으로 나가지 않습니다; 계정도, 클라우드도, 추적도 없습니다
- **자동 백업** -- 서버가 스마트 보존 일정에 따라 백업을 생성하며, 최대 50개 파일을 유지합니다
- **전체 내보내기 / 가져오기** -- 모든 데이터(북마크, 메모, 설정, 아이콘, 배경)를 하나의 JSON, Markdown, 또는 HTML 파일로 다운로드할 수 있습니다
- **휴지통과 실행 취소** -- 실수로 삭제했나요? Ctrl+Z를 누르면 복구됩니다 (10단계 실행 취소 스택 -- 최대 10번 연속으로 되돌릴 수 있습니다)
- **localhost 전용** -- 서버는 `127.0.0.1`(내 컴퓨터)에서만 응답합니다; 같은 네트워크의 다른 사람도 접근할 수 없습니다
- **외부 의존성 제로** -- 서버는 Node.js의 내장 기능만 사용합니다; 추가로 다운로드하거나 업데이트할 것이 없습니다
- **오프라인 지원** -- 인터넷 연결 없이도 대시보드가 작동합니다 (날씨 제외)
- **프로필** -- 서로 다른 대시보드 설정을 저장하고 전환할 수 있습니다
- **백업 목록 및 복원 UI** -- 모든 백업을 날짜와 크기와 함께 탐색하고, 클릭 한 번으로 원하는 백업을 복원할 수 있습니다
- **가져오기 미리보기** -- 가져오기를 적용하기 전에, 정확히 몇 개의 북마크, 메모, 기타 항목이 가져와질지 요약을 확인할 수 있습니다
- **탭 동기화** -- 대시보드를 여러 브라우저 탭에서 열어 두면, 변경 사항이 탭 간에 즉시 동기화됩니다 (BroadcastChannel API 사용)

## 🧰 요구 사항

> "요구 사항"이란 시작하기 전에 준비해야 할 것들을 말합니다.

| 무엇이 필요한가요 | 왜 필요한가요 | 어떻게 구하나요 |
|------|-----|---------------|
| **컴퓨터** | Windows 10/11, macOS, 또는 Linux | 이미 가지고 계실 겁니다 |
| **구글 크롬** (또는 아무 브라우저) | 대시보드를 보기 위해 | [크롬 다운로드](https://www.google.com/chrome/) |
| **인터넷 연결** | 최초 설치 시에만 필요 | 설치 후에는 오프라인으로 작동합니다 |
| **Node.js** | 대시보드를 뒤에서 실행하는 작은 프로그램 | **Windows:** 설치 파일이 자동으로 다운로드합니다. **Mac/Linux:** [nodejs.org](https://nodejs.org)에서 설치 |

> **Node.js란 무엇인가요?** 대시보드를 뒤에서 움직이는 작은 엔진(engine)이라고 생각하세요. 대시보드가 "자동차"라면, Node.js는 자동차를 움직이게 하는 "엔진"입니다. 뒤에서 조용히 돌아가며 메모리를 아주 적게 사용합니다 (약 15 MB -- 브라우저 탭 하나보다 적습니다). Windows에서는 설치 파일(`setup_windows.bat`)이 Node.js의 포터블 사본을 자동으로 다운로드하므로 직접 설치할 필요가 없습니다. 시스템에 Node.js가 이미 설치되어 있으면 그것을 사용하고, 없으면 이 프로젝트 전용 사본을 다운로드합니다.

## 📥 설치

### 🪟 Windows (가장 쉬움 -- 코딩 불필요)

> **소요 시간:** 약 2~5분.

**1단계: 프로젝트 다운로드**

이 프로젝트 폴더를 다운로드하여 원하는 곳에 놓으세요.

예시: `C:\Users\내이름\Dashboard`

> 바탕화면, 문서 폴더, 어디든 괜찮습니다. 다만 `C:\Windows`나 `C:\Program Files` 같은 시스템 폴더는 피하세요. 참고: 이 폴더를 나중에 삭제하면 대시보드도 함께 사라지므로 안전한 곳에 보관하세요.

**2단계: 설치 실행**

폴더를 열고 **`setup_windows.bat`**이라는 파일을 찾으세요. 톱니바퀴 같은 아이콘의 작은 스크립트 파일입니다. **더블클릭**하세요.

검은색 창(명령 프롬프트(Command Prompt) -- 컴퓨터에 텍스트로 지시를 내리는 방식)이 나타나며 진행 상황이 표시됩니다:
1. Node.js(엔진(engine))가 이미 컴퓨터에 있는지 확인합니다
2. 없으면 포터블 사본을 자동으로 다운로드합니다 (~30 MB -- 최초 1회만)
3. Node.js가 시스템 전체에 이미 설치되어 있으면 그것을 사용합니다
4. 대시보드 서버가 백그라운드에서 시작됩니다 (보이는 창 없음)
5. 자동 시작이 등록되어 컴퓨터를 켤 때마다 자동으로 실행됩니다

**"Setup Complete!"**라는 메시지가 나타날 때까지 기다리세요 -- 그 후에 검은색 창을 닫아도 됩니다.

**3단계: 크롬에서 열기**

크롬을 열고 **주소창**(브라우저 창 맨 위에 있는 긴 텍스트 상자, 보통 웹사이트 주소가 표시되는 곳)에 다음 주소를 입력하세요:

```
http://localhost:1111
```

그리고 **Enter** 키를 누르세요.

> **`localhost:1111`은 무슨 뜻인가요?** "localhost"는 "이 컴퓨터"라는 뜻입니다 -- 인터넷 대신 자기 컴퓨터를 들여다보라고 크롬에게 말하는 것이죠. "1111"은 **포트**(port) 번호입니다 (건물의 방 번호(door number)라고 생각하세요 -- 설정 섹션에서 더 자세히 설명합니다).

**설치 완료!** 이제부터 컴퓨터를 켤 때마다 대시보드가 자동으로 시작됩니다 -- 팝업도, 추가 창도 없습니다.

1. 기존 데이터가 있는지 확인하고 안전 백업을 먼저 생성합니다 (`safety-before-setup.json`)
2. `node/` 폴더에 Node.js 포터블 사본을 다운로드합니다 (아직 없고 시스템 Node.js도 없는 경우)
3. 이전에 실행 중이던 대시보드 서버를 중지합니다
4. 로그인 시 대시보드가 자동으로 시작되도록 등록합니다 (관리자 권한 불필요)
5. 서버를 백그라운드에서 조용히 시작합니다 (보이는 창 없음)

시스템 전체에 아무것도 설치하지 않습니다. 자동 시작 항목 하나 외에는 Windows 설정을 수정하지 않습니다. 모든 것이 프로젝트 폴더 안에 머무릅니다.

### 🍎 Mac

> **소요 시간:** 약 3분. 몇 가지 명령을 입력해야 합니다.

**1단계: Node.js 설치**

[nodejs.org](https://nodejs.org)로 이동하여 웹사이트의 큰 초록색 버튼을 클릭하세요. 안내에 따라 설치를 진행합니다.

**2단계: 터미널 열기**

**터미널**(Terminal)이라는 앱을 열어야 합니다 (클릭 대신 타이핑으로 컴퓨터에 말을 거는 텍스트 기반 창입니다):

- 키보드에서 `Cmd + Space`을 누르세요 (Spotlight 검색이 열립니다)
- `Terminal`을 입력하고 Enter를 누르세요
- 텍스트 프롬프트가 있는 창이 나타납니다 -- 여기에 명령을 입력합니다

**3단계: 프로젝트 폴더로 이동**

다음 명령을 입력하고 Enter를 누르세요. 터미널에게 프로젝트를 놓아둔 폴더로 "이동하라"고 지시하는 것입니다:

```bash
cd ~/Dashboard
```

> `~/Dashboard`를 실제로 프로젝트 폴더를 놓아둔 경로로 바꾸세요. `cd` 명령은 "change directory"(디렉토리 변경, 즉 폴더로 이동)라는 뜻입니다. `~` 기호는 홈 폴더를 의미합니다.

**4단계: 설치 파일에 실행 권한 부여**

```bash
chmod +x setup_mac.sh
```

> `chmod` 명령은 설치 파일에 실행 권한을 부여합니다. Mac에서는 보안상 이 단계가 필요합니다.

**5단계: 설치 실행**

```bash
./setup_mac.sh
```

> `./` 접두사는 "현재 폴더에 있는 이 파일을 실행하라"는 뜻입니다.

**6단계: 크롬에서 열기**

크롬을 열고 다음 주소로 이동하세요:

```
http://localhost:1111
```

**7단계: 완료!**

대시보드는 로그인할 때마다 자동으로 시작됩니다. 이제 터미널을 닫아도 됩니다.

### 🐧 Linux

1. [nodejs.org](https://nodejs.org) 또는 패키지 관리자에서 Node.js를 설치하세요 (예: `sudo apt install nodejs`)
2. **터미널**(명령을 입력하는 텍스트 창 -- 대부분의 Linux 시스템에서 `Ctrl+Alt+T`로 엽니다)을 여세요

<details>
<summary>systemd로 자동 시작 설정하기 (클릭해서 읽기)</summary>

**systemd**는 대부분의 Linux 시스템에 내장된 도구로, 컴퓨터가 부팅될 때 프로그램을 자동으로 시작할 수 있습니다. 대시보드가 자동으로 시작되도록 systemd "서비스"를 생성하세요:

```bash
sudo tee /etc/systemd/system/dashboard.service << EOF
[Unit]
Description=Chrome Starting Page Dashboard
After=network.target

[Service]
Type=simple
User=$USER
WorkingDirectory=$HOME/Dashboard
ExecStart=$(which node) server.js
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable dashboard
sudo systemctl start dashboard
```

> `$HOME/Dashboard`를 실제 프로젝트 폴더 경로로 바꾸세요 (다른 위치인 경우).

</details>

### 포트 번호 변경

> **"포트"란 무엇인가요?** 건물의 방 번호(door number)라고 생각하세요. 컴퓨터에는 이런 "방"이 수천 개 있고, 각 프로그램이 서로 다른 방을 사용합니다. 대시보드는 기본적으로 방 번호 1111을 사용합니다. 다른 프로그램이 이미 그 방을 쓰고 있다면, 다른 번호를 골라야 합니다.

대시보드는 기본적으로 포트 `1111`에서 실행됩니다. 변경하려면:

| 방법 | 어떻게 |
|--------|-----|
| **대시보드에서** (가장 쉬움) | 설정 > 데이터 > 서버 관리 > 새 포트 입력 > "변경" 클릭 |
| **파일 더블클릭** | `set-port.bat` 실행, 번호 입력 (예: `8080`), Enter |
| **텍스트 파일 편집** | 프로젝트 폴더의 `port.conf` 파일을 만들거나 편집하고, 안에 번호만 적기 |
| **명령줄** | `node server.js 8080` 실행 |

> 포트를 변경한 후에는 크롬 시작 페이지 주소도 업데이트하세요 (예: `http://localhost:8080`).

## 💾 데이터 안전

### 내 데이터는 어디에 저장되나요?

모든 데이터는 프로젝트 안의 `data/` 폴더에 저장됩니다. 이 폴더는 설치, 재시작, 업데이트 스크립트에 의해 **절대 삭제되지 않습니다**. 외부 서버로 전송되는 것도 전혀 없습니다.

마치 책상 서랍에 일기장(diary)을 넣어 두는 것과 같습니다 -- 내 컴퓨터의 내 폴더에 그대로 머물러 있고, 나만 볼 수 있습니다.

### 자동 백업

서버가 스마트 보존 일정에 따라 자동으로 백업을 생성합니다. 별도로 할 일은 없습니다.

| 시간 범위 | 유지되는 백업 수 |
|-----------|--------------------------|
| 최근 7일 | 모든 백업 유지 |
| 8일~30일 | 하루에 백업 1개 |
| 30일 초과 | 자동 삭제 |
| 최대 합계 | 백업 파일 50개 |

백업은 `data/backups/`에 저장됩니다.

### 백업 목록 및 복원

대시보드에서 직접 모든 백업을 탐색하고 복원할 수 있습니다:

1. **설정 > 데이터**로 이동하세요
2. 날짜와 파일 크기가 포함된 **모든 백업 목록**이 표시됩니다
3. 원하는 백업 옆의 **"복원"**을 클릭하면 해당 시점으로 대시보드를 되돌릴 수 있습니다

실수로 무언가를 삭제했거나, 설정을 변경하고 되돌리고 싶을 때 유용합니다.

|--------|-------|
| **JSON으로 내보내기** (모든 것을 하나의 파일로 저장) | `S` 누르기 > 데이터 탭 > "내보내기" 클릭 > 모든 북마크, 메모, 설정, 아이콘, 배경, 프로필이 담긴 파일이 다운로드됩니다 |
| **Markdown으로 내보내기** | `S` 누르기 > 데이터 탭 > Markdown 형식 선택 |
| **HTML로 내보내기** | `S` 누르기 > 데이터 탭 > HTML 형식 선택 |
| **가져오기** (파일에서 복원) | `S` 누르기 > 데이터 탭 > "가져오기" 클릭 > 백업 파일 선택 > 정확히 몇 개의 북마크, 메모, 할 일, 기타 항목이 가져와질지 **미리보기** 확인 > 확인을 눌러 적용 |
| **빠른 복원 (드래그 앤 드롭)** | 백업 JSON 파일을 대시보드 페이지 위에 끌어다 놓으세요 |

> **가져오기 미리보기:** 가져오기가 적용되기 전에, 각 항목 유형별 개수를 보여주는 요약 화면이 표시됩니다 (예: "북마크 42개, 메모 5개, 할 일 12개"). 이를 통해 현재 데이터를 덮어쓰기 전에 파일이 올바른지 확인할 수 있습니다.

> 내보내기 파일을 USB 드라이브나 클라우드 저장소(구글 드라이브 등)에 저장해 두면 더욱 안전합니다. 마치 여행 가방(travel bag)에 소중한 물건을 넣어 두는 것처럼요.

### 다른 컴퓨터로 이동

1. 기존 컴퓨터에서: 데이터를 **내보내기** (설정 > 데이터 > 내보내기)
2. 다운로드된 파일을 새 컴퓨터로 복사 (USB, 이메일, 클라우드 드라이브 -- 무엇이든 됩니다)
3. 새 컴퓨터에서: 대시보드를 설치한 뒤 파일을 **가져오기** (설정 > 데이터 > 가져오기)
4. 완료 -- 대시보드가 똑같이 보입니다

> **대안 방법:** 기존 컴퓨터의 `data/` 폴더 전체를 복사하여 새 컴퓨터의 프로젝트 폴더 안 같은 위치에 붙여넣으세요.

### 재설치 시 어떻게 되나요?

`setup_windows.bat`을 다시 실행하면:

1. 현재 데이터의 **안전 백업**을 먼저 생성합니다 (백업 폴더에 `safety-before-setup.json`으로 저장)
2. **기존 데이터를 모두 그대로 유지합니다** -- 아무것도 삭제되지 않습니다
3. 부팅 시 대시보드가 실행되도록 **자동 시작을 재등록**합니다
4. 최신 파일로 **서버를 재시작**합니다

북마크, 메모, 할 일, 설정 모두 그대로 있습니다. 아무것도 사라지지 않습니다.

## 🗑️ 제거 (깨끗한 삭제)

### Windows (권장)

1. **`uninstall.bat`**을 더블클릭하세요
2. 자동으로 다음을 수행합니다:
   - 서버 중지
   - Windows 레지스트리 및 시작프로그램 폴더에서 자동 시작 제거
   - 데이터를 유지할지 삭제할지 선택 (직접 선택)
   - 로그 파일 및 임시 파일 정리
3. 이후 프로젝트 폴더 전체를 안전하게 삭제할 수 있습니다
4. 크롬에서 `chrome://settings/onStartup`으로 이동하여 localhost 항목을 제거하세요

시스템에 흔적이 남지 않습니다. 다른 프로그램에 영향을 주지 않습니다. 이 프로그램은 시스템 파일을 수정하지 않으므로, 폴더를 삭제하면 깨끗하게 제거됩니다.

### Mac

가장 쉬운 방법: 터미널을 열고 프로젝트 폴더로 이동한 뒤 제거 스크립트를 실행하세요:

```bash
cd ~/Dashboard
chmod +x uninstall_mac.sh
./uninstall_mac.sh
```

스크립트가 서버를 중지하고, 자동 시작을 제거하고, 데이터를 유지할지 삭제할지 물어봅니다.

<details>
<summary>수동 제거 (직접 하고 싶은 경우)</summary>

터미널을 열고 다음 명령을 실행하세요:

```bash
launchctl unload ~/Library/LaunchAgents/com.dashboard.startpage.plist
rm ~/Library/LaunchAgents/com.dashboard.startpage.plist
```

그런 다음 프로젝트 폴더를 삭제하세요.

</details>

### Linux

터미널을 열고 다음 명령을 실행하세요:

```bash
sudo systemctl stop dashboard
sudo systemctl disable dashboard
sudo rm /etc/systemd/system/dashboard.service
sudo systemctl daemon-reload
```

그런 다음 프로젝트 폴더를 삭제하세요.

## ❓ 문제 해결

<details>
<summary><strong>페이지가 열리지 않습니다 ("이 사이트에 연결할 수 없음")</strong></summary>

1. `restart.bat`을 더블클릭하여 서버를 재시작하세요
2. 주소가 `http://localhost:1111`인지 확인하세요 -- "https"가 아닌 "http" ("s" 없이)
3. 포트를 변경했다면 올바른 번호를 사용하세요 (예: `http://localhost:8080`)
4. Node.js가 작동하는지 확인하세요: 명령 프롬프트를 열고 (Windows 키를 누르고 `cmd`를 입력, Enter), `node -v`를 입력한 뒤 Enter를 누르세요. `v22.19.0` 같은 버전 번호가 보여야 합니다
   - 버전 번호가 보이면 Node.js는 정상입니다 -- `setup_windows.bat`을 다시 실행해 보세요
   - "명령으로 인식되지 않습니다"라고 나오면 `setup_windows.bat`을 다시 실행하여 Node.js를 재다운로드하세요

</details>

<details>
<summary><strong>"포트가 이미 사용 중입니다" 오류</strong></summary>

컴퓨터의 다른 프로그램이 이미 해당 포트 번호(방 번호(door number))를 사용하고 있습니다.

해결 방법:
1. `set-port.bat`을 더블클릭하고 다른 번호를 선택하세요 (예: `2222`, `8080`, 또는 `9999`)
2. `restart.bat`을 더블클릭하여 서버를 재시작하세요
3. 크롬 시작 페이지 주소를 새 포트에 맞게 업데이트하세요 (예: `http://localhost:8080`)

</details>

<details>
<summary><strong>북마크 아이콘이 표시되지 않습니다</strong></summary>

아이콘은 인터넷에서 자동으로 다운로드됩니다. 누락된 경우:
- 아이콘을 처음 로드하려면 인터넷 연결이 필요합니다
- 일부 웹사이트는 아이콘을 제공하지 않습니다 -- 대신 사이트 이름의 첫 두 글자가 표시됩니다
- 시스템이 포기하기 전에 4가지 다른 소스를 자동으로 시도합니다
- 인터넷이 연결되어 있는데도 아이콘이 나타나지 않으면 잠시 기다리세요. 자동으로 재시도됩니다

</details>

<details>
<summary><strong>데이터가 저장되지 않습니다 ("저장 실패" 메시지)</strong></summary>

1. 서버가 중지되었을 수 있습니다 -- `restart.bat`을 더블클릭하세요
2. 프로젝트 폴더 안에 `data/` 폴더가 있는지 확인하세요
3. 폴더가 없으면 `data`라는 이름의 빈 폴더를 만드세요
4. 폴더가 "읽기 전용"으로 설정되어 있지 않은지 확인하세요 (프로젝트 폴더를 우클릭 > 속성 > "읽기 전용" 체크 해제)

</details>

<details>
<summary><strong>업데이트 후 페이지가 깨져 보입니다</strong></summary>

브라우저가 이전의 캐시된 파일을 사용하고 있습니다. 해결 방법:
1. 크롬에서 `Ctrl+Shift+Delete`를 누르세요
2. "캐시된 이미지 및 파일"을 체크하고 기간을 "지난 1시간"으로 설정하세요
3. "데이터 삭제"를 클릭하세요
4. 페이지를 새로고침하세요 (`Ctrl+R` 또는 `F5`)

</details>

<details>
<summary><strong>재부팅 후 서버가 자동 시작되지 않습니다</strong></summary>

- **Windows:** `setup_windows.bat`을 다시 실행하여 자동 시작을 재등록하세요
- **Mac:** `./setup_mac.sh`를 다시 실행하세요
- **Linux:** `sudo systemctl enable dashboard`를 실행하세요

</details>

<details>
<summary><strong>서버를 재시작하고 싶습니다</strong></summary>

두 가지 방법:
- **대시보드에서:** 설정 > 데이터 > "서버 재시작" 버튼
- **파일에서:** `restart.bat` 더블클릭

두 방법 모두 대시보드 서버만 안전하게 중지합니다. 컴퓨터의 다른 프로그램은 절대 영향받지 않습니다.

</details>

<details>
<summary><strong>시작할 때 검은 창이 잠깐 나타납니다</strong></summary>

정상적으로는 서버가 백그라운드에서 보이지 않게 시작되며 창이 나타나지 않아야 합니다. 컴퓨터를 시작할 때마다 짧은 검은 창(명령 프롬프트)이 보인다면:

1. `setup_windows.bat`을 다시 실행하세요
2. 서버가 보이는 창 없이 백그라운드에서 실행되도록 자동 시작을 올바르게 재등록합니다

</details>

**방법 1 (권장):**

**방법 2 (수동 복사):**

</details>

<details>
<summary><h2>🔬 기술 세부 사항 (궁금한 분들을 위해)</h2></summary>

개발자이거나 내부 작동 방식이 궁금하신 분들을 위한 내용입니다:

### 아키텍처

```
Chrome 브라우저  <--HTTP-->  로컬 Node.js 서버 (포트 1111)  <--읽기/쓰기-->  디스크의 JSON 파일
                                  (server.js)                                     (data/*.json)
```

서버는 Node.js 내장 모듈만 사용하는 단일 파일 HTTP 서버입니다. npm 패키지는 사용하지 않습니다. 프론트엔드는 빌드 도구나 트랜스파일 없이 작동하는 단일 페이지 애플리케이션입니다.

| 항목 | 세부 사항 |
|------|--------|
| **서버** | Node.js 내장 모듈만 사용 (`http`, `fs`, `path`, `zlib`, `crypto`) -- `npm install` 불필요 |
| **프론트엔드** | 순수 HTML + CSS + JavaScript -- 프레임워크 없음, 빌드 단계 없음, 트랜스파일 없음 |
| **기본 포트** | 1111 (`port.conf`, 명령줄 인수, 대시보드 UI, 또는 `set-port.bat`으로 변경 가능) |
| **바인딩** | `127.0.0.1` 전용 -- 네트워크의 다른 컴퓨터에서 접근할 수 없습니다 |
| **데이터 저장** | `data/` 디렉토리의 로컬 JSON 파일 |

### 성능

| 항목 | 세부 사항 |
|------|--------|
| **압축** | HTML, CSS, JS, JSON, SVG, webmanifest에 Gzip 적용 |
| **캐싱** | SHA-256 ETag + 5분 TTL의 인메모리 파일 캐시 |
| **오프라인 지원** | Service Worker가 핵심 파일을 미리 캐시 |
| **메모리 사용량** | 유휴 시 약 15 MB |
| **최대 요청 본문** | 10 MB |
| **오류 로그 순환** | 로그는 10 MB로 제한되며 자동 순환 |
| **부팅 순서** | 3단계: 즉시 렌더링 > 지연 초기화 > 30초 후 백그라운드 검사 |

### 보안

| 항목 | 세부 사항 |
|------|--------|
| **경로 탐색 방지** | 서버가 프로젝트 폴더 외부 파일 접근을 차단 |
| **파일명 정제** | 악의적인 파일명이 필터링됩니다 |
| **쓰기 잠금** | 타임아웃 기반 쓰기 잠금으로 데이터 무결성 보장 |
| **localhost 전용 바인딩** | `127.0.0.1` 전용 -- 같은 네트워크의 다른 컴퓨터도 접근할 수 없습니다 |
| **프로세스 관리** | PID 파일 (`.server.pid`)로 다른 프로그램에 영향 없이 안전하게 재시작 |
| **포터블 모드** | Windows 설치가 Node.js를 로컬 `node/` 폴더에 다운로드하여 시스템 Node.js와 완전 독립 |
| **사용자 정의 CSS XSS 필터** | 사용자가 제공한 사용자 정의 CSS를 정제하여 크로스 사이트 스크립팅 공격을 방지 |
